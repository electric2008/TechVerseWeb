Cybersecurity

แฮกเกอร์คนเดียวเกือบพลิกสัญญาณถ่ายทอดสดฟุตบอลโลกได้ทั้งหมด แค่สมัครเป็นตัวแทนนักเตะ

TV
TechVerse Thailand · 17 มิถุนายน 2569 · อ่าน 2 นาที

ลองจินตนาการดูนะครับ คุณกำลังดูฟุตบอลโลกอยู่ดีๆ แล้วจู่ๆ สัญญาณทีวีทั่วโลกก็เปลี่ยนไปเป็นอะไรบางอย่างที่ไม่ใช่ฟุตบอล ฟังดูเหมือนหนัง แต่เรื่องนี้เกือบเกิดขึ้นจริงแล้ว และที่น่าตกใจกว่านั้นคือช่องโหว่ที่เปิดโอกาสให้ทำแบบนี้ได้ มันเริ่มต้นจากการสมัครบัญชีธรรมดาๆ บนเว็บไซต์ของ FIFA เท่านั้นเอง

นักวิจัยด้านความปลอดภัยไซเบอร์ที่ใช้ชื่อออนไลน์ว่า BobDaHacker เปิดเผยในบล็อกโพสต์ว่าเธอค้นพบช่องโหว่สำคัญในระบบภายในของ FIFA ระหว่างการแข่งขันฟุตบอลโลก จุดเริ่มต้นของเรื่องนี้ไม่ได้ซับซ้อนเลย เธอแค่สมัครบัญชีในฐานะตัวแทนนักเตะบนแพลตฟอร์มจดทะเบียนตัวแทนอย่างเป็นทางการของ FIFA ซึ่งเป็นระบบที่ใครก็สมัครได้ทั่วไป แต่ปัญหาคือ API ที่อยู่เบื้องหลังระบบของ FIFA มีข้อบกพร่องร้ายแรง นั่นคือมันไม่ได้ตรวจสอบจริงๆ ว่าผู้ใช้คนนั้นมีสิทธิ์เข้าถึงข้อมูลส่วนนั้นหรือเปล่า พูดง่ายๆ ก็คือ ระบบเปิดประตูให้เดินเข้าไปได้เลย ทั้งที่ไม่ควรมีกุญแจในมือ

ด้วยช่องโหว่นี้ BobDaHacker สามารถเข้าถึงแพลตฟอร์มภายในหลายระบบของ FIFA รวมถึงระบบที่ใช้ควบคุมสัญญาณถ่ายทอดสดที่ส่งไปยังทีวีทั่วโลก และระบบที่แสดงข้อมูลบนหน้าจอของผู้บรรยายในสนาม เธอบอกว่าถ้าเป็นคนที่มีเจตนาร้าย ก็สามารถควบคุมกล้องทุกตัวพร้อมกันได้ในคราวเดียว และอาจเปลี่ยนสัญญาณทีวีระหว่างฟุตบอลโลกให้กลายเป็นอะไรก็ได้ตามใจชอบ ซึ่งเธอใช้คำว่าอาจเอาไปเปิดวิดีโอ rickroll ให้คนดูทั่วโลกดูแทนฟุตบอลได้เลย นี่ไม่ใช่แค่มุกตลก แต่สะท้อนให้เห็นว่าผลกระทบที่อาจเกิดขึ้นจากช่องโหว่นี้มันใหญ่มากแค่ไหน

สิ่งที่น่าสังเกตในเรื่องนี้มีสองอย่างครับ อย่างแรกคือ BobDaHacker รายงานช่องโหว่นี้ให้ FIFA ทราบในคืนวันอังคารตามเวลาญี่ปุ่น และ FIFA ก็แก้ไขปัญหาได้ภายในไม่กี่ชั่วโมงหลังจากนั้น ฟังดูดีใช่ไหมครับ แต่ประเด็นคืออย่างที่สอง FIFA ไม่เคยตอบรับหรือยืนยันกับนักวิจัยคนนี้เลยว่าได้รับรายงานแล้ว และเมื่อ TechCrunch ติดต่อไปขอความเห็น FIFA ก็ไม่ได้ตอบกลับเช่นกัน พฤติกรรมแบบนี้ในแวดวงความปลอดภัยไซเบอร์เรียกว่าไม่ค่อยโอเคเลย เพราะนักวิจัยที่ค้นพบช่องโหว่และรายงานอย่างมีจรรยาบรรณควรได้รับการตอบรับขั้นต่ำสุด ไม่ใช่เงียบใส่

สำหรับคนไทยที่ติดตามข่าวนี้ มีประเด็นที่น่าคิดต่อสองเรื่องครับ เรื่องแรกคือเรื่องของ API Security ซึ่งเป็นปัญหาที่เกิดขึ้นบ่อยมากในระบบดิจิทัลทั่วโลก รวมถึงระบบของไทยด้วย หลายๆ ครั้งที่เราได้ยินข่าวข้อมูลรั่วไหล ต้นเหตุมักมาจากจุดนี้แหละ คือระบบไม่ได้ตรวจสอบสิทธิ์ผู้ใช้อย่างจริงจัง เรื่องที่สองคือเรื่องของการรายงานช่องโหว่อย่างมีความรับผิดชอบ หรือที่เรียกว่า Responsible Disclosure ในต่างประเทศมีวัฒนธรรมนี้ค่อนข้างชัดเจน นักวิจัยพบช่องโหว่ก็รายงานให้บริษัทแก้ก่อนค่อยเปิดเผยสู่สาธารณะ แต่บริษัทก็ควรตอบรับและให้เครดิตด้วย ซึ่งในกรณีนี้ FIFA ไม่ได้ทำ

เรื่องนี้เตือนใจเราว่าแม้แต่งานอีเวนต์ระดับโลกที่มีงบประมาณมหาศาล ระบบไอทีเบื้องหลังก็อาจมีรูรั่วที่คาดไม่ถึงได้เสมอ และบางทีช่องโหว่ที่อันตรายที่สุดก็ไม่ได้ซับซ้อนอะไรเลยครับ แค่ระบบลืมตรวจว่าคนที่กำลังเข้ามาควรมีสิทธิ์อยู่ที่นั่นจริงๆ หรือเปล่า ถ้าอยากรู้ว่าช่องโหว่ประเภทนี้ยังมีอยู่ในระบบอื่นๆ อีกแค่ไหน บอกได้เลยว่าเรื่องนี้ยังไม่จบแค่นี้แน่นอน 🔐
